# إدارة الأصول (Asset Management) في أنظمة HR

## 📋 ما هي إدارة الأصول؟

إدارة الأصول هي نظام لتتبع وإدارة جميع الممتلكات والأصول التي توزعها الشركة على الموظفين أو تستخدمها في العمل.

---

## 🎯 الأهداف الرئيسية

1. **تتبع الأصول**: معرفة أين كل أصل ومن يستخدمه
2. **إدارة التوزيع**: توزيع الأصول على الموظفين
3. **الصيانة**: جدولة وتتبع صيانة الأصول
4. **الاسترجاع**: استرجاع الأصول عند إنهاء الخدمة أو الاستبدال
5. **التكلفة**: حساب تكلفة الأصول وإهلاكها

---

## 📦 أنواع الأصول الشائعة

### 1. **الأصول التقنية**
- 💻 أجهزة اللابتوب
- 📱 الهواتف الذكية
- 🖥️ أجهزة الكمبيوتر المكتبية
- ⌨️ لوحات المفاتيح والفأرة
- 🖨️ الطابعات
- 📶 أجهزة الراوتر/الشبكة
- 🎧 سماعات الرأس

### 2. **الأصول المكتبية**
- 🪑 الكراسي المكتبية
- 🗄️ المكاتب
- 📚 خزائن الملفات
- 💡 الإضاءة

### 3. **الأصول المتنقلة**
- 🚗 السيارات
- 🏍️ الدراجات النارية
- 🚲 الدراجات

### 4. **الأصول الأخرى**
- 🔑 المفاتيح
- 🎒 حقائب العمل
- 📱 أجهزة التابلت
- 📷 الكاميرات

## 🔧 الميزات الأساسية المطلوبة

### 1. **إدارة الأصول (Assets)**
```
- إضافة أصل جديد
- تعديل معلومات الأصل
- حذف الأصل
- تصنيف الأصول (نوع، فئة)
- معلومات الأصل:
  * اسم الأصل
  * رقم التسلسلي/الباركود
  * الشركة المصنعة
  * الموديل
  * تاريخ الشراء
  * التكلفة
  * الحالة (جديد، مستخدم، معطل، مفقود)
  * الموقع (فرع، قسم)
```

### 2. **توزيع الأصول (Asset Assignment)**
```
- توزيع أصل على موظف
- تاريخ التوزيع
- تاريخ الاسترجاع المتوقع
- حالة التوزيع (نشط، مسترجع)
- ملاحظات
```

### 3. **استرجاع الأصول (Asset Return)**
```
- استرجاع أصل من موظف
- تاريخ الاسترجاع
- حالة الأصل عند الاسترجاع
- ملاحظات الاسترجاع
- التحقق من سلامة الأصل
```

### 4. **صيانة الأصول (Asset Maintenance)**
```
- جدولة الصيانة
- سجل الصيانة
- تكلفة الصيانة
- حالة الصيانة (مجدولة، قيد التنفيذ، مكتملة)
- تاريخ الصيانة القادمة
```

### 5. **تتبع الأصول (Asset Tracking)**
```
- معرفة من يستخدم كل أصل
- تاريخ التوزيع
- سجل التوزيعات السابقة
- حالة الأصل الحالية
```

### 6. **التقارير (Reports)**
```
- تقرير الأصول الموزعة
- تقرير الأصول المتاحة
- تقرير الأصول المعطلة
- تقرير الأصول المفقودة
- تقرير الصيانة
- تقرير التكلفة
```

## 📊 مثال عملي

### سيناريو 1: توزيع لابتوب جديد
```
1. إضافة لابتوب جديد في النظام:
   - اسم: "Dell Latitude 5520"
   - رقم تسلسلي: "DL5520-12345"
   - تاريخ الشراء: 2025-01-15
   - التكلفة: 5000 ريال
   - الحالة: جديد

2. توزيع اللابتوب على موظف:
   - الموظف: أحمد محمد
   - تاريخ التوزيع: 2025-01-20
   - ملاحظات: "لابتوب جديد للموظف الجديد"

3. النظام يسجل:
   - الأصل مرتبط بالموظف
   - الحالة: مستخدم
   - تاريخ آخر تحديث
```

### سيناريو 2: استرجاع أصل
```
1. الموظف ينهي خدمته
2. HR يسترجع الأصل:
   - تاريخ الاسترجاع: 2025-12-08
   - حالة الأصل: جيد
   - ملاحظات: "تم الاسترجاع بنجاح"

3. النظام:
   - يغير حالة الأصل إلى "متاح"
   - يزيل الربط مع الموظف
   - يحفظ سجل الاسترجاع
```

### سيناريو 3: صيانة أصل
```
1. جدولة صيانة:
   - الأصل: طابعة HP
   - نوع الصيانة: تنظيف وصيانة دورية
   - تاريخ الصيانة: 2025-12-15
   - التكلفة المتوقعة: 200 ريال

2. بعد الصيانة:
   - تحديث السجل
   - تسجيل التكلفة الفعلية
   - جدولة الصيانة القادمة
```

## 🗄️ هيكل قاعدة البيانات المقترح

### جدول `assets` (الأصول)
```sql
- id
- asset_code (كود الأصل)
- name (اسم الأصل)
- name_ar (اسم الأصل بالعربي)
- category (الفئة)
- type (النوع)
- manufacturer (الشركة المصنعة)
- model (الموديل)
- serial_number (الرقم التسلسلي)
- purchase_date (تاريخ الشراء)
- purchase_cost (تكلفة الشراء)
- current_value (القيمة الحالية)
- status (الحالة: available, assigned, maintenance, damaged, lost)
- location (الموقع: branch_id, department_id)
- warranty_expiry (انتهاء الضمان)
- notes (ملاحظات)
- created_by
- timestamps
- soft_deletes
```

### جدول `asset_assignments` (توزيع الأصول)
```sql
- id
- asset_id
- employee_id
- assigned_date (تاريخ التوزيع)
- expected_return_date (تاريخ الاسترجاع المتوقع)
- actual_return_date (تاريخ الاسترجاع الفعلي)
- assignment_status (حالة التوزيع: active, returned)
- condition_on_assignment (حالة الأصل عند التوزيع)
- condition_on_return (حالة الأصل عند الاسترجاع)
- notes (ملاحظات)
- assigned_by (من وزع)
- returned_by (من استرجع)
- created_by
- timestamps
- soft_deletes
```

### جدول `asset_maintenances` (صيانة الأصول)
```sql
- id
- asset_id
- maintenance_type (نوع الصيانة)
- scheduled_date (تاريخ الصيانة المجدول)
- actual_date (تاريخ الصيانة الفعلي)
- cost (التكلفة)
- description (الوصف)
- status (الحالة: scheduled, in_progress, completed, cancelled)
- next_maintenance_date (تاريخ الصيانة القادمة)
- service_provider (مزود الخدمة)
- notes (ملاحظات)
- created_by
- timestamps
- soft_deletes
```

## 🎨 واجهة المستخدم المقترحة

### 1. **صفحة قائمة الأصول**
- جدول بجميع الأصول
- فلترة حسب: الحالة، النوع، الموقع، الموظف
- بحث
- أزرار: إضافة، تعديل، حذف، توزيع، صيانة

### 2. **صفحة إضافة/تعديل أصل**
- نموذج كامل بمعلومات الأصل
- رفع صور للأصل (اختياري)
- QR Code/Barcode للأصل

### 3. **صفحة توزيع أصل**
- اختيار الموظف
- تاريخ التوزيع
- ملاحظات
- حالة الأصل

### 4. **صفحة استرجاع أصل**
- معلومات الموظف
- معلومات الأصل
- حالة الأصل عند الاسترجاع
- ملاحظات

### 5. **صفحة صيانة أصل**
- جدولة الصيانة
- سجل الصيانة السابقة
- التكلفة

### 6. **Dashboard**
- إحصائيات:
  * إجمالي الأصول
  * الأصول الموزعة
  * الأصول المتاحة
  * الأصول قيد الصيانة
  * الأصول المعطلة
- رسوم بيانية

## 🔗 التكامل مع النظام الحالي

### 1. **التكامل مع الموظفين**
- عند إنهاء خدمة موظف، إشعار HR لاسترجاع الأصول
- عرض الأصول الموزعة على الموظف في ملفه الشخصي

### 2. **التكامل مع الفروع والأقسام**
- ربط الأصول بالفروع والأقسام
- تقارير حسب الموقع

### 3. **التكامل مع إنهاء الخدمة**
- عند إنهاء الخدمة، فحص الأصول الموزعة
- إجبار استرجاع الأصول قبل إكمال العملية

---

## 💡 ميزات إضافية متقدمة

1. **QR Code/Barcode**: مسح ضوئي لتتبع الأصول
2. **إشعارات**: تنبيهات انتهاء الضمان، مواعيد الصيانة
3. **الإهلاك**: حساب إهلاك الأصول تلقائياً
4. **التكامل مع المحاسبة**: ربط الأصول بالحسابات المالية
5. **التقارير المتقدمة**: تقارير مفصلة بتكلفة الأصول

---

## 📈 الفوائد

1. **توفير المال**: تجنب فقدان الأصول
2. **الكفاءة**: معرفة سريعة لمن يستخدم ماذا
3. **المساءلة**: كل أصل مرتبط بموظف
4. **الصيانة**: صيانة دورية تمنع الأعطال
5. **التخطيط**: معرفة متى نحتاج لشراء أصول جديدة

---

## ✅ الخلاصة

إدارة الأصول هي نظام شامل لتتبع وإدارة جميع ممتلكات الشركة التي توزع على الموظفين أو تستخدم في العمل. تساعد في:
- توفير المال
- تحسين الكفاءة
- المساءلة
- الصيانة الوقائية
- التخطيط
